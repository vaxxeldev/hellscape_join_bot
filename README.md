# Flood Games Join Bot

Telegram-бот для вступления в закрытое игровое комьюнити по Genshin Impact и Honkai: Star Rail.

Бот проверяет подписку на каналы, принимает анкету, отправляет ее в админ-чат, выдает личную invite-ссылку после одобрения и обрабатывает `chat_join_request`. Отдельно поддержана бронь ролей.

## Стек

- Node.js 20+
- TypeScript
- Telegraf
- SQLite через `better-sqlite3`
- `.env` для всех токенов, ID и ссылок

## Установка

```bash
npm install
cp .env.example .env
```

Заполните `.env` реальными значениями:

```env
BOT_TOKEN=
LIFE_CHANNEL_ID=
INFO_CHANNEL_ID=
MAIN_CHAT_ID=
ADMIN_CHAT_ID=
RULES_URL=
LIFE_CHANNEL_URL=
INFO_CHANNEL_URL=
ROLE_POST_GENSHIN_URL=
ROLE_POST_HSR_URL=
CODE_WORD=
DEVELOPER_ID=
OWNER_ID=
CO_OWNER_IDS=
SENIOR_ADMIN_IDS=
JUNIOR_ADMIN_IDS=
INVITE_EXPIRE_HOURS=24
DATABASE_URL=file:./data/flood_games.sqlite
TELEGRAM_API_ROOT=https://api.telegram.org
TELEGRAM_PROXY_URL=
LAUNCH_RETRY_SECONDS=15
```

`DEVELOPER_ID` — отдельная техническая роль разработчика бота с теми же правами доступа, что и у админов, но без привязки к ролям администрации сообщества.
Списки админов указываются через запятую: `111,222,333`.

Ссылки на посты ролей задаются через `ROLE_POST_GENSHIN_URL` и `ROLE_POST_HSR_URL`.
Алиасы и сокращения ролей лежат в `roles/aliases.json`.
ID premium emoji лежат в `premium_emoji.json`; если ID пустой, бот использует обычный emoji fallback.

## Запуск

Разработка:

```bash
npm run dev
```

Сборка и запуск:

```bash
npm run build
npm start
```

При первом запуске бот применит миграции из `migrations/`.

Если запуск пишет `ETIMEDOUT` на `api.telegram.org`, это проблема сетевого доступа до Telegram Bot API. Бот будет повторять запуск каждые `LAUNCH_RETRY_SECONDS` секунд.

Варианты:

- указать HTTP/HTTPS-прокси в `TELEGRAM_PROXY_URL`, например `http://127.0.0.1:7890`;
- использовать локальный Telegram Bot API server и указать его в `TELEGRAM_API_ROOT`.

## Права бота в Telegram

Бот должен быть:

- администратором лайф-канала;
- администратором инфо-канала / канала правил;
- администратором основного закрытого чата;
- участником или администратором админ-чата.

В основном закрытом чате боту нужны права:

- приглашать пользователей через ссылки;
- принимать заявки на вступление;
- отклонять заявки на вступление.

Для приватных каналов проверка подписок через `getChatMember` обычно требует, чтобы бот был администратором этих каналов.

## Пользовательские команды

- `/start` — открыть приветствие и меню.
- `/status` — проверить статус последней анкеты.
- `/rules` — получить ссылку на правила.
- `/reserve` — забронировать роль.
- `/my_reserve` — посмотреть активную бронь.
- `/cancel` — отменить заполнение анкеты или брони.
- `/help` — помощь.

## Админ-команды

- `/admin` — админ-панель.
- `/stats` — статистика.
- `/applications` — последние анкеты.
- `/app ID` — открыть конкретную анкету.
- `/ban USER_ID|@username` — запретить пользователю подавать анкеты.
- `/unban USER_ID|@username` — снять запрет (и сбросить счётчик входов).
- `/reservations` — список активных броней.
- `/expire_reserve ID` — вручную закрыть бронь.
- `/use_reserve ID` — отметить бронь использованной.
- `/reload` — перечитать `.env` без перезапуска, кроме `BOT_TOKEN`.
- `/help_admin` — список админ-команд.

## Как работает вступление

1. Пользователь нажимает «Подать анкету».
2. Бот проверяет подписку на лайф-канал и инфо-канал через Telegram Bot API.
3. Если подписки есть, бот задает вопросы анкеты.
4. Анкета сохраняется в SQLite со статусом `pending`.
5. Админы получают карточку анкеты с кнопками.
6. При одобрении бот повторно проверяет подписки и кодовое слово.
7. Бот создает личную invite-ссылку с `creates_join_request: true` и сроком действия.
8. Когда пользователь отправляет заявку по ссылке, бот сверяет ссылку, владельца, срок, статус анкеты и подписки.
9. Админы принимают или отклоняют заявку кнопками в админ-чате.
10. После решения ссылка отзывается, чтобы ее нельзя было использовать повторно.

## Бронь роли

Бронь не выдает ссылку в основной чат. Это отдельная заявка, которую админы могут одобрить или отклонить. Истекшие брони закрываются:

- при запуске бота;
- периодически по `RESERVATION_EXPIRE_CHECK_HOURS`.

## Безопасность

- ID чатов, каналов и админов не хардкодятся.
- Callback-кнопки доступны только разрешенным админам.
- Callback-data содержит только короткие ID; все данные заново читаются из БД.
- Анкету нельзя одобрить дважды.
- Invite-ссылка связана с `user_id` и `application_id`.
- Ссылка отзывается после использования, отказа или невалидной заявки.
- Основная идентификация идет по `telegram_id`, а не по username.
- Ошибки Telegram API логируются через `pino`.
